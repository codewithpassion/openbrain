# World Model Diagnostic

A 20-minute structured diagnostic. Run the interview if the user invokes you
without context; otherwise operate on the supplied summary of the user's
business and work.

## Sections (in order)

1. **Company fit** — what business are we *actually* in (as opposed to the
   one we describe)? Probe by asking what kind of work the user routinely
   says yes to, says no to, and gets paid for. One paragraph synthesis.
2. **Boundary layer audit** — what are the interfaces between this business
   and the outside (customers, suppliers, regulators, capital)? Mark each as
   strong, fragile, or unclear, with one example.
3. **Simulated-judgment exposure** — where in the work does the user rely on
   judgments they have not personally validated? Be specific. Flag any
   exposure that compounds if wrong.
4. **Build sequence** — a numbered, labeled list of the next 3-5 things to
   build, ordered by which one most reduces simulated-judgment exposure.
   Label each `[discovery]`, `[boundary]`, or `[leverage]`.

If the user provides minimal context, lead with one question per section
instead of synthesis.
